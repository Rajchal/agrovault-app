# Build & Distribution Guide

## Download & Install

### For Development Testing

**Option 1: Expo Go (Easiest)**
1. Install Expo Go app on your phone
   - iOS: https://apps.apple.com/app/expo-go/id1054959411
   - Android: https://play.google.com/store/apps/details?id=host.exp.exponent
2. Run: `npm start`
3. Scan QR code in terminal

**Option 2: Local Emulator**
```bash
npm run ios     # iOS Simulator (macOS)
npm run android # Android Emulator
```

## Build for App Stores

### iOS App Store

**Prerequisites:**
- Apple Developer Account ($99/year)
- macOS with Xcode 14+
- TestFlight access (free)

**Steps:**

1. **Create provisioning profile:**
   ```bash
   npm install -g eas-cli
   eas login
   eas credentials
   ```

2. **Build:**
   ```bash
   eas build --platform ios
   ```

3. **Upload to TestFlight:**
   - `eas submit --platform ios`
   - Or submit from App Store Connect manually

4. **Submit to App Store:**
   - TestFlight testing → approval → release

**Timeline:** 24-48 hours for App Store review

---

### Google Play Store

**Prerequisites:**
- Google Play Developer Account ($25 one-time)
- Signing key (auto-generated by Expo)

**Steps:**

1. **Build APK/AAB:**
   ```bash
   eas build --platform android
   # Creates App Bundle (.aab) for Play Store
   ```

2. **Submit to Play Store:**
   ```bash
   eas submit --platform android
   # Or upload .aab manually to Play Console
   ```

3. **Configure:**
   - Fill app listing
   - Set pricing/countries
   - Add screenshots

4. **Release:**
   - Internal Testing → Closed Testing → Open Beta → Production
   - Gradual rollout (5% → 50% → 100%)

**Timeline:** 2-4 hours for initial approval, faster for updates

## Direct APK Distribution (Android)

For side-loading without Play Store:

```bash
# Build APK locally
eas build --platform android --local

# Or use Android Gradle directly
cd android
./gradlew assembleRelease
```

Share `.apk` file directly:
- Email, Google Drive, GitHub Releases
- Users install with: `adb install app.apk`

**⚠️ Warning:** Users must enable "Unknown Sources" in Settings

## Manual Builds (Advanced)

### iOS (Xcode)

```bash
# Generate native Xcode project
expo prebuild --platform ios --clean

# Build in Xcode
open ios/agrovault.xcworkspace

# Or use CLI
xcodebuild -workspace ios/agrovault.xcworkspace \
  -scheme agrovault -configuration Release \
  -derivedDataPath ios/build
```

### Android (Gradle)

```bash
# Generate native Android project
expo prebuild --platform android --clean

cd android
./gradlew assembleRelease

# Signed APK: build/outputs/apk/release/app-release.apk
```

## Code Signing

### iOS Certificate

```bash
# Auto-managed by Expo
eas credentials

# Select or create certificate
# Passphrase will be stored securely
```

### Android Keystore

```bash
# Auto-managed by Expo
eas credentials

# Or create custom:
keytool -genkey -v -keystore my-key.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias my-key-alias
```

## Over-the-Air Updates

Update app content without app store resubmission:

```bash
# Install EAS Update
npm install expo-updates

# Make code changes
# Then deploy:
eas update --channel production

# Users auto-download next time app opens
```

**Benefits:**
- Instant bug fixes
- No App Store review
- Gradual rollouts

## Performance Optimization Before Release

### 1. Minification
```bash
# Included in production builds automatically
eas build --platform ios --release-channel production
```

### 2. Bundle Size
```bash
# Analyze
npm run analyze

# Target <50MB for iOS, <100MB for Android
```

### 3. App Icon & Splash
- Add to `assets/` folder
- Update in `app.json`:
```json
{
  "icon": "./assets/icon.png",
  "splash": { "image": "./assets/splash.png" }
}
```

### 4. Update App Version
```json
{
  "version": "1.0.1",
  "ios": { "buildNumber": "2" },
  "android": { "versionCode": 2 }
}
```

## Troubleshooting Builds

| Error | Solution |
|-------|----------|
| `Certificate expired` | Regenerate: `eas credentials` |
| `Pod install failed` | `cd ios && pod repo update && pod install` |
| `Build timeout` | Increase timeout or try local build: `eas build --local` |
| `Insufficient storage` | `eas build --platform android --local` |
| `Review rejected` | Check App Store guidelines, resubmit with notes |
